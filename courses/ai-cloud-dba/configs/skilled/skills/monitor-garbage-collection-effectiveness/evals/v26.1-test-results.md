### ⚠️ PARTIAL: Protected Timestamps Monitoring

#### Test 1.4: Protected Timestamps (Lines 148-163)
**Query**:
```sql
SELECT
  id,
  timestamp,
  (now() - timestamp)::INTERVAL AS age,
  meta_type,
  CASE
    WHEN meta_type = 'jobs' THEN 'Backup/Changefeed Job'
    WHEN meta_type = 'schema_change' THEN 'Schema Change'
    ELSE meta_type
  END AS description
FROM crdb_internal.kv_protected_ts_records
ORDER BY timestamp;
```
**Result**: ⚠️ **PARTIAL** - Works but column name wrong
```
ERROR: column "timestamp" does not exist
```

**v26.1 Corrected Query**:
```sql
SET allow_unsafe_internals = true;

SELECT
  id,
  ts,  -- NOT "timestamp"
  -- ts is DECIMAL nanoseconds since epoch
  (extract(epoch from now()) - (ts::DECIMAL / 1000000000)) / 3600 AS age_hours,
  meta_type,
  CASE
    WHEN meta_type = 'jobs' THEN 'Backup/Changefeed Job'
    WHEN meta_type = 'schema_change' THEN 'Schema Change'
    ELSE meta_type
  END AS description
FROM crdb_internal.kv_protected_ts_records
ORDER BY ts;
```

**Important Notes**:
1. Column name is `ts` not `timestamp`
2. `ts` is a DECIMAL representing nanoseconds since epoch, not TIMESTAMPTZ
3. Age calculation must convert nanoseconds to a time duration
4. May encounter cluster-specific errors (e.g., stale descriptor references)

**Tested Output** (with error):
```

uuid  | 1772766697054787000.0000000000 | schedules
ERROR: internal error: descriptor not found
```
